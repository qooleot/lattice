# Lattice Mermaid Docs (Diagram Projection) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the mermaid diagram projection (statecharts, per-context domain diagram, cross-context context map), the structural language growth it needs (qualified refs, `contextMap` files), and the whole-language reference docs — per the approved spec [`docs/superpowers/specs/2026-07-05-lattice-mermaid-docs-design.md`](../specs/2026-07-05-lattice-mermaid-docs-design.md).

**Architecture:** Each spec dir stays one context/one session/one canonical `spec.lat`. A workspace index `specs/context-map.lat` (new `contextMap` top-level form in the same Langium grammar) declares contexts + relationships. Pure mermaid emitters (`src/emit/mermaid/`) render from the AST; `writeProjections` writes per-spec diagrams on every `emit`/`apply`/`sync`; a workspace compile (new `docs` command + an `apply` hook) renders the context map. Qualified refs (`Catalog.Plan`) are structural only — loudly excluded from all invariant/solver machinery.

**Tech Stack:** TypeScript ESM (`.js` import suffixes), Langium 3.5 (grammar → `npx langium generate`, generated code gitignored), vitest + fast-check, `mermaid` + `jsdom` (new dev-deps, syntax gate only).

## Global Constraints

Copied from the spec + repo conventions; every task implicitly includes these.

- Working dir for all commands: `lattice/` (`cd lattice` first). Tests: `npx vitest run <file>`; whole suite `npm test`; typecheck `npm run typecheck`.
- After ANY edit to `src/parse/lat.langium`, run `npx langium generate` before typecheck/tests. `src/parse/generated/` is gitignored — never commit it.
- The `.lat` file is the git-facing canonical spec (slice-3 P1). No new authored JSON artifacts. The AST is an in-memory intermediate.
- Diagnostics never throw; they return `{ code, message, ... }` objects. New diagnostic codes in this slice: `wrong-file-kind`, `cross-context-ref-unsupported`, `duplicate-context`, `unknown-relationship-endpoint`, `self-relationship`, `unknown-exposed-type`, `uncovered-cross-context-ref`, `context-name-mismatch`, `missing-member`, `workspace-invalid`.
- `//` comments are banned in `.lat` (parse error); `///` is the only comment form and round-trips into `doc` fields.
- Naming (P8, warning-level): contexts/enums/entities/aggregates/events/contextMap names PascalCase; fields/states/regions/transitions/invariants/enum values camelCase.
- Round-trip identity `parse ∘ print = id` is property-tested; every printable construct must round-trip.
- `RESERVED_WORDS` (`src/ast/reserved.ts`) must stay in two-way lockstep with quoted grammar keywords — enforced by the grammar-sync test in `test/parse/parse.test.ts`.
- Generated files carry headers: `<!-- generated by lattice; do not edit -->` (`.md`) / `%% generated by lattice; do not edit` (`.mmd`).
- Every ` ```lat ` block in `docs/language/` must be a complete parseable file; deliberately-invalid examples use plain fences.
- Commits: conventional style used in this repo, e.g. `feat(lattice): …`, `docs(language): …`, ending with `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- Mermaid emitters are pure (`AST → string`), no filesystem access. All file I/O stays in `cli.ts` / `engine/workspace.ts`.

entry LatFile:

ContextMapDecl:

```ts
import type { Diagnostic } from './invariant.js';

export type Role = 'openHost' | 'publishedLanguage' | 'anticorruption' | 'conformist';
export type RelationshipKind = 'upstreamDownstream' | 'partnership' | 'sharedKernel';
export interface ContextRef { name: string; path: string }   // path always resolved (explicit or default)
export interface Relationship {
  kind: RelationshipKind;
  left: string; right: string;            // upstreamDownstream: left = upstream, right = downstream
  upstreamRoles?: Role[]; downstreamRoles?: Role[];   // upstreamDownstream only
  exposes?: string[];
  doc?: string;
}
export interface ContextMapModel { name: string; contexts: ContextRef[]; relationships: Relationship[]; doc?: string }

export const defaultPath = (contextName: string): string =>
  contextName.charAt(0).toLowerCase() + contextName.slice(1);

export function validateContextMap(m: ContextMapModel): Diagnostic[];
// codes: invalid-name / reserved-word (names), duplicate-context, self-relationship,
// unknown-relationship-endpoint (endpoint not in contexts[])
```

- Produces (`fromLangium.ts`): `export function loadContextMapText(text: string): ContextMapLoadResult` where

```ts
export type ContextMapLoadResult =

```

`warnings` carries `naming-convention` for non-PascalCase map/context names. Mirror of Task 3's guard: a *context* file fed to `loadContextMapText` → `wrong-file-kind`.

export function loadContextMapText(text: string): ContextMapLoadResult {
  const parsed = parseLat(text);
  if (!parsed.ok) return parsed;
  const file = parsed.cst;
  if (!file.map)
    return { ok: false, diagnostics: [{ code: 'wrong-file-kind', line: 1, col: 1,
      message: 'expected a contextMap file, got a context — the workspace map lives in context-map.lat' }] };
  const m = file.map;
  const map: ContextMapModel = {
    name: m.name,
    contexts: m.contains.map(c => ({ name: c.name, path: c.path ?? defaultPath(c.name) })),
    relationships: m.relationships.map(r => {
      const rel: Relationship = {
        kind: r.kind === 'upstream' ? 'upstreamDownstream' : r.kind as 'partnership' | 'sharedKernel',
        left: r.left, right: r.right,
      };
      if (r.upstreamRoles.length) rel.upstreamRoles = [...r.upstreamRoles] as Role[];
      if (r.downstreamRoles.length) rel.downstreamRoles = [...r.downstreamRoles] as Role[];
      if (r.exposes.length) rel.exposes = [...r.exposes];
      const d = joinDocs([...r.docs]); if (d) rel.doc = d;
      return rel;
    }),
  };
  const topDoc = joinDocs([...file.docs]); if (topDoc) map.doc = topDoc;
  const diags = validateContextMap(map).map(d =>
    ({ code: d.code, message: d.at ? `${d.message} (at ${d.at})` : d.message, line: 1, col: 1 }));
  if (diags.length) return { ok: false, diagnostics: diags };
  const warnings: ParseDiagnostic[] = [];
  const PASCAL = /^[A-Z][A-Za-z0-9]*$/;
  if (!PASCAL.test(map.name)) warnings.push({ code: 'naming-convention', ...at(file.map),
    message: `contextMap '${map.name}' should be PascalCase (spec P8)` });
  for (const c of m.contains) if (!PASCAL.test(c.name)) warnings.push({ code: 'naming-convention',
    ...at(c), message: `context '${c.name}' should be PascalCase (spec P8)` });

export function contextMapToCode(map: ContextMapModel): string {
  const out: string[] = [];
  doc(map.doc, '', out);
  out.push(`contextMap ${map.name} {`);
  for (const c of map.contexts)
    out.push(`  contains ${c.name}${c.path === defaultPath(c.name) ? '' : ` from "${c.path}"`}`);
  for (const r of map.relationships) {
    out.push('');
    doc(r.doc, '  ', out);
    const head = r.kind === 'upstreamDownstream'
      ? `${r.left} upstream of ${r.right}`
      : `${r.left} ${r.kind === 'partnership' ? 'partnership' : 'sharedKernel'} with ${r.right}`;
    out.push(`  ${head} {`);
    if (r.upstreamRoles?.length) out.push(`    upstream roles ${r.upstreamRoles.join(', ')}`);
    if (r.downstreamRoles?.length) out.push(`    downstream roles ${r.downstreamRoles.join(', ')}`);
    if (r.exposes?.length) out.push(`    exposes ${r.exposes.join(', ')}`);
    out.push('  }');
  }
  out.push('}');
  return out.join('\n') + '\n';
}

import type { AggregateDef, Region } from '../../ast/domain.js';

/** stateDiagram-v2 for one region: [*]→initial, one labeled edge per declared transition,
 *  @terminal states →[*]. Names are grammar-ID-constrained, hence mermaid-safe (spec §5). */
export function machineToMermaid(agg: AggregateDef, region: Region): string {
  const out = ['stateDiagram-v2', `  [*] --> ${region.initial}`];
  for (const t of (agg.machine?.transitions ?? []).filter(t => t.region === region.name))

    out.push(`  ${s.name} --> [*]`);
  return out.join('\n') + '\n';
}
